## 项目链接

- [ibelick/ui-skills](https://github.com/ibelick/ui-skills)
- [UI Skills 官网](https://www.ui-skills.com/)

## 这是什么

`UI Skills` 是一个面向 Design Engineer 的 skill 目录，重点不在 UI 组件本身，而在把“怎么把界面做得更好”整理成一组可复用的 AI skill。

目前覆盖的方向主要包括：

- UI 基线规范检查
- 无障碍审查与修复
- 动效性能排查
- 前端界面设计生成
- 设计实验与交互细节打磨

## 为什么值得收录

- 这类资源更像 AI agent 的能力目录，不是单个 prompt，也不是单一教程，后续复用价值更高。
- 主题很聚焦，补的是前端里常常被忽略的部分：审美、规范、可访问性、动效体验。
- 如果后面要整理自己的前端 / 设计类 skill，这个目录可以直接当参考样板和选题池。

## 当前看到的几个代表方向

基于官网目录，目前比较有代表性的条目包括：

- `baseline-ui`：偏 UI 规范、一致性和组件层面的基础审查
- `fixing-accessibility`：偏无障碍问题排查与修复
- `fixing-motion-performance`：偏动画卡顿、布局抖动和性能治理
- `frontend-design`：偏高质量页面和组件设计生成
- `make-interfaces-feel-better`：偏微交互、排版和视觉细节精修

## 适合回看的场景

- 给 AI coding agent 增加前端设计审查能力
- 做 landing page、营销页、作品集页面时找切入点
- 补齐 UI / UX / a11y / motion 这类非业务逻辑能力
- 反向整理自己的设计工程 skill 清单

## 我的备注

这个项目更像“设计工程 skill 索引”，不是传统意义上的组件库或设计灵感站。

如果之后我自己也想沉淀类似能力，最值得参考的不是具体文案，而是它的组织方式：按问题类型拆 skill，让 AI 在不同设计场景下调用不同能力，而不是用一个大而全的前端 prompt 混着做。
